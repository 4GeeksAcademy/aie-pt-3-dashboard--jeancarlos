# CONTEXT
Debes construir un dashboard para una influencer que promociona productos mediante múltiples plataformas digitales.

La influencer publica contenido en:

- Instagram
- TikTok
- YouTube

Necesita consolidar toda la información en un único dashboard para analizar el rendimiento de sus campañas y colaboraciones.

Productos promocionados:

- Producto A → 50 €
- Producto B → 120 €
- Producto C → 80 €

Comisión:

- 15% por cada venta generada.

El dashboard debe responder preguntas como:

- ¿Cuánto dinero generan las campañas?
- ¿Cuánto dinero generan las comisiones?
- ¿Qué producto vende más?
- ¿Qué producto genera más ingresos?
- ¿Qué plataforma tiene mejor rendimiento?
- ¿Cuál tiene mejor conversión?
- ¿Cuál tiene mejor engagement?
- ¿Qué campañas funcionan mejor?

### Estructura del Dashboard

Mostrar indicadores principales:

- Ingresos Totales
- Comisiones Totales
- Conversiones Totales
- Alcance Total
- Engagement Rate
- Conversion Rate
- CTR
- ROAS

#### Bloque Intermedio — Drivers

- Funnel de Conversión

Mostrar información detallada:

- Tabla de Productos

- Alertas
- Rankings
- Oportunidades de Mejora

# ACCEPTANCE CRITERIA

## HTML Semántico
Utilizar correctamente:

- header
- nav
- main
- section
- article
- aside
- footer
- table
- thead
- tbody

Mantener una jerarquía clara:

- Un único h1
- h2 para secciones principales
- h3 para subsecciones

Evitar divs innecesarios.

---

## Tailwind CSS
Usar exclusivamente clases utility de Tailwind CSS v4.

Utilizar correctamente:

- Flexbox
- Grid
- Gap
- Padding
- Margin
- Responsive utilities

Evitar:

- CSS redundante
- Inline styles
- Dimensiones fijas innecesarias

---

## Responsive Design
Implementar Mobile First.

Breakpoints mínimos:

- Mobile
- sm
- md
- lg

El dashboard debe visualizarse correctamente en:

- Smartphones
- Tablets
- Escritorio

No debe existir:

- Scroll horizontal
- Contenido roto
- Solapamiento de elementos

---

## Componentización
Crear patrones visuales reutilizables:

- KPI Cards
- Dashboard Sections
- Driver Cards
- Data Tables
- Alert Components
- Filter Components

Los componentes similares deben compartir estructura y estilo.

---

## Accesibilidad
Cumplir estándares WCAG AA.

Incluir:

- Etiquetas descriptivas
- Jerarquía correcta de encabezados
- Contraste adecuado
- Navegación por teclado
- ARIA cuando sea necesario

---
